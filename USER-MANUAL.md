# 테이블오더 서비스 — 사용자 매뉴얼

## 접속 정보

| 구분 | URL |
|------|-----|
| 관리자 페이지 | http://localhost/admin |
| 고객 메뉴 화면 | http://localhost/customer/menu |

> 서비스가 실행 중이어야 합니다. `nerdctl compose up -d --build`로 시작하세요.

---

## 1. 관리자 사용법

### 1.1 로그인

1. http://localhost/admin/login 접속
2. 아래 기본 계정으로 로그인

| 필드 | 값 |
|------|-----|
| 매장 식별자 | default |
| 사용자명 | admin |
| 비밀번호 | admin123 |

- 로그인 성공 시 대시보드로 이동합니다.
- 비밀번호 5회 연속 실패 시 15분간 계정이 잠깁니다.
- 세션은 16시간 유지됩니다.

---

### 1.2 메뉴 관리

좌측 사이드바에서 "메뉴 관리"를 선택합니다.

#### 카테고리

- 카테고리 추가: 카테고리명 입력 후 등록
- 카테고리 수정: 기존 카테고리명 클릭 후 편집
- 카테고리 삭제: 삭제 시 소속 메뉴는 "미분류"로 자동 이동
- 순서 변경: 드래그 앤 드롭으로 노출 순서 조정 후 저장

> "미분류" 카테고리는 시스템 기본 카테고리로 삭제할 수 없습니다.

#### 메뉴

- 메뉴 등록: "메뉴 추가" 버튼 → 메뉴명, 가격, 설명, 카테고리 선택, 이미지 업로드(선택)
- 메뉴 수정: 메뉴 항목 클릭 → 정보 수정 후 저장
- 메뉴 삭제: 삭제 버튼 → 확인 팝업 → 삭제
- 순서 변경: 드래그 앤 드롭으로 노출 순서 조정 후 저장

이미지 제한사항:
- 허용 형식: jpg, jpeg, png, gif, webp
- 최대 크기: 5MB

---

### 1.3 테이블 설정

좌측 사이드바에서 "테이블 관리"를 선택합니다.

#### 새 테이블 등록 (고객 태블릿 준비)

1. 테이블 번호(정수)와 비밀번호를 입력
2. "설정" 버튼 클릭
3. 16시간 유효한 세션이 자동 생성됩니다

#### 고객 태블릿에 접속 설정

테이블 설정 후, 해당 태블릿 브라우저에서 아래 URL로 접속합니다:

```
http://localhost/customer/menu?store=default&table={테이블번호}&password={비밀번호}
```

예시 (테이블 1번, 비밀번호 table1234):
```
http://localhost/customer/menu?store=default&table=1&password=table1234
```

- 최초 1회 접속하면 인증 정보가 태블릿에 저장됩니다.
- 이후 브라우저를 열면 자동 로그인되어 메뉴 화면이 표시됩니다.
- 세션 만료(16시간) 시 관리자가 테이블을 재설정해야 합니다.

#### 테이블 재설정

기존 테이블 번호로 다시 설정하면:
- 이전 세션이 자동 종료됩니다
- 새 세션이 생성됩니다
- 태블릿에서 다시 위 URL로 접속해야 합니다

---

### 1.4 주문 모니터링 (대시보드)

좌측 사이드바에서 "대시보드"를 선택합니다.

- 테이블별 카드가 그리드로 표시됩니다.
- 각 카드에 총 주문액과 주문 건수가 보입니다.
- 신규 주문이 들어오면 해당 카드가 강조 표시되고 알림이 뜹니다 (실시간).

#### 주문 상태 변경

1. 테이블 카드 클릭 → 주문 목록 패널 열기
2. 각 주문의 상태 변경 버튼 사용:

| 현재 상태 | 가능한 변경 | 버튼 |
|-----------|------------|------|
| 대기중 | → 준비중 | "준비 시작" |
| 준비중 | → 완료 | "준비 완료" |
| 준비중 | → 대기중 | "대기로 복귀" |
| 완료 | 변경 불가 | — |

#### 주문 삭제

- 주문 목록에서 삭제 버튼 클릭 → 확인 팝업 → 삭제
- 삭제된 주문은 목록에서 사라지고 총 주문액이 재계산됩니다.

#### 이용 완료 (테이블 정리)

고객이 떠난 후:

1. 주문 목록 패널에서 "이용 완료" 버튼 클릭
2. 확인 팝업에서 "확인" 클릭
3. 현재 주문이 과거 이력으로 이동됩니다
4. 테이블 카드가 리셋됩니다 (주문 0건, 총액 0원)
5. 새 고객을 받으려면 테이블을 재설정해야 합니다

---

### 1.5 과거 주문 내역

좌측 사이드바에서 "과거 내역"을 선택합니다.

- 테이블별 과거 주문 이력을 조회할 수 있습니다.
- 날짜 필터로 특정 기간의 주문만 볼 수 있습니다.
- 각 주문의 메뉴 목록, 수량, 금액, 이용 완료 시각이 표시됩니다.

---

## 2. 고객(태블릿) 사용법

### 2.1 메뉴 탐색

- 태블릿 브라우저를 열면 메뉴 화면이 자동으로 표시됩니다.
- 좌측 카테고리를 선택하면 해당 카테고리 메뉴만 표시됩니다.
- 메뉴 카드를 터치하면 상세 정보(설명, 이미지)를 볼 수 있습니다.

### 2.2 장바구니

1. 메뉴 상세에서 수량 선택 후 "장바구니 추가" 터치
2. 화면 하단 장바구니 바에 수량과 총 금액이 표시됩니다
3. 장바구니 바를 터치하면 장바구니 페이지로 이동합니다

장바구니 페이지에서:
- 수량 변경: +/- 버튼
- 항목 삭제: 삭제 버튼
- 전체 비우기: "장바구니 비우기" 버튼
- 브라우저를 새로고침해도 장바구니 내용이 유지됩니다

### 2.3 주문하기

1. 장바구니 페이지에서 "주문하기" 터치
2. 주문 확인 페이지에서 내역 확인 후 "주문 확정" 터치
3. 주문 성공 시:
   - 주문 번호가 표시됩니다
   - 장바구니가 자동으로 비워집니다
   - 5초 후 메뉴 화면으로 자동 이동합니다 (또는 "메뉴로 돌아가기" 터치)
4. 주문 실패 시:
   - 에러 메시지가 표시됩니다
   - 장바구니 내용은 유지됩니다

### 2.4 주문 내역 확인

- 화면에서 "주문 내역" 메뉴를 터치합니다.
- 현재 세션에서 주문한 내역만 표시됩니다.
- 각 주문의 상태(대기중/준비중/완료)를 확인할 수 있습니다.
- 관리자가 상태를 변경하면 실시간으로 반영됩니다.

---

## 3. 운영 흐름 요약

```
[관리자] 메뉴/카테고리 등록
         ↓
[관리자] 테이블 설정 (번호 + 비밀번호)
         ↓
[관리자] 태블릿에 고객 URL 접속 설정
         ↓
[고객]   메뉴 탐색 → 장바구니 → 주문
         ↓
[관리자] 대시보드에서 실시간 주문 확인
         ↓
[관리자] 주문 상태 변경 (대기중 → 준비중 → 완료)
         ↓
[고객]   주문 상태 실시간 확인
         ↓
[관리자] 고객 퇴장 후 "이용 완료" 처리
         ↓
[관리자] 새 고객을 위해 테이블 재설정
```

## 4. 자주 묻는 질문

**Q: 태블릿에서 "세션 만료" 메시지가 뜹니다.**
A: 세션 유효 시간(16시간)이 지났습니다. 관리자가 테이블 관리에서 해당 테이블을 재설정한 후, 태블릿에서 고객 URL로 다시 접속하세요.

**Q: 관리자 계정이 잠겼습니다.**
A: 비밀번호 5회 연속 실패 시 15분간 잠깁니다. 15분 후 다시 시도하세요.

**Q: 이용 완료 후 주문이 사라졌습니다.**
A: 이용 완료 처리 시 현재 주문은 과거 이력으로 이동됩니다. "과거 내역" 메뉴에서 확인할 수 있습니다.

**Q: 완료된 주문의 상태를 되돌릴 수 있나요?**
A: 아니요. "완료" 상태는 최종 상태로 변경할 수 없습니다. "준비중" 상태에서는 "대기로 복귀"가 가능합니다.

**Q: 메뉴 가격을 변경하면 기존 주문에 영향이 있나요?**
A: 없습니다. 주문 시점의 메뉴명과 가격이 스냅샷으로 저장되므로, 이후 메뉴 변경은 기존 주문에 영향을 주지 않습니다.

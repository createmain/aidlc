## 1. 폼 검증 규칙 (인라인 에러)

| 필드 | 규칙 | 에러 메시지 |
|------|------|-------------|

| password | 필수 | "비밀번호를 입력해 주세요" |

### 1.2 메뉴 등록/수정 폼

| 필드 | 규칙 | 에러 메시지 |

| image | jpg/jpeg/png/gif/webp | "지원하지 않는 이미지 형식입니다" |
| image | 최대 5MB | "이미지 크기는 5MB 이하여야 합니다" |

### 1.3 카테고리 등록/수정

| 필드 | 규칙 | 에러 메시지 |
|------|------|-------------|
| name | 필수, 1~50자 | "카테고리명을 입력해 주세요" / "50자 이내" |

### 1.4 테이블 초기 설정 폼

| 필드 | 규칙 | 에러 메시지 |
|------|------|-------------|

## 3. 주문 상태 전이 규칙 (UI)

### 3.1 허용 전이 (api-specification.md 기준)

| 현재 상태 | 허용 전이 | 표시 버튼 |
|-----------|-----------|-----------|
| pending | preparing | "준비 시작" |
| preparing | completed, pending | "준비 완료", "대기로 복귀" |
| completed | (없음 — 종착 상태) | 버튼 없음 |

### 3.2 상태 뱃지

| 상태 | 색상 | 텍스트 |
|------|------|--------|
| pending | 주황 | "대기중" |
| preparing | 파랑 | "준비중" |
| completed | 초록 | "완료" |

### 3.3 삭제/아카이브 주문
- is_deleted=true → 목록에서 미표시
- is_archived=true → 현재 주문 목록에서 제외

## 4. 인증 규칙

### 4.1 관리자 JWT

| 항목 | 값 |
|------|-----|
| 저장 위치 | localStorage 'admin_token' |
| 만료 | 16시간 (서버 설정) |
| 만료 확인 | 페이지 로드 시 + API 401 시 |
| 만료 동작 | localStorage 삭제 → LoginPage |
| API 요청 | Authorization: Bearer {token} |

### 4.2 테이블 토큰

| 항목 | 값 |
|------|-----|
| 저장 위치 | localStorage 'table_token' |
| 추가 저장 | localStorage 'table_id', 'session_id' |
| 만료 | 세션 남은 시간 (최대 16시간) |
| 만료 확인 | 앱 마운트 시 + API 401 시 |
| 만료 동작 | 에러 화면: "세션 만료. 관리자에게 문의하세요" |

## 5. 에러 처리 규칙

### 5.1 표시 패턴

| 에러 유형 | 표시 방식 | 자동 사라짐 |
|-----------|-----------|:-----------:|
| 네트워크/500 | 토스트 (error) | 5초 |
| 비즈니스 에러 (400, 409) | 토스트 (warning) | 5초 |
| 폼 검증 | 인라인 (필드 하단) | 수정 시 |
| 인증 (401) | 자동 로그아웃 + 리다이렉트 | - |
| 계정 잠금 (423) | 인라인 (로그인 폼) | 수정 시 |

### 5.2 Backend 에러 코드 매핑

| error 코드 | Frontend 처리 |
|------------|---------------|
| VALIDATION_ERROR | 토스트 + details 있으면 인라인 매핑 |
| AUTH_ERROR | 인라인 에러 (로그인 폼) |
| FORBIDDEN | 토스트: "권한이 없습니다" |
| NOT_FOUND | 토스트: "데이터를 찾을 수 없습니다" |
| STATE_ERROR | 토스트: 서버 message 그대로 표시 |
| ACCOUNT_LOCKED | 인라인: "계정 잠금. {retryAfter}분 후 재시도" |
| INTERNAL_ERROR | 토스트: "서버 오류가 발생했습니다" |

### 5.3 토스트 규칙

| 항목 | 값 |
|------|-----|
| 최대 동시 | 3개 |
| 자동 사라짐 | success: 3초, error/warning: 5초, info: 4초 |
| 위치 | 화면 상단 우측 |
| 수동 닫기 | 항상 가능 (X 버튼) |

## 6. Realtime(SSE) 규칙

### 6.1 이벤트 처리

| 이벤트 | UI 동작 |
|--------|---------|
| new-order | 테이블 데이터 갱신 + 토스트("테이블 {번호} 새 주문") + 카드 하이라이트 |
| order-status-changed | 주문 상태 뱃지 갱신 |
| order-deleted | 주문 제거 + newTableTotal로 총액 갱신 |
| session-completed | 테이블 카드 리셋 (주문 0, 총액 0) |

### 6.2 하이라이트 규칙

| 항목 | 동작 |
|------|------|
| 트리거 | new-order 이벤트 |
| 표시 | 카드 테두리/배경 강조 |
| 해제 | 관리자가 해당 카드 클릭 시 |
| 토스트 | 자동 사라짐 (5초) |

### 6.3 연결 상태 인디케이터

| 상태 | 표시 |
|------|------|
| connected | 초록 점 + "연결됨" |
| disconnected | 빨간 점 + "연결 끊김" |
| reconnecting | 노란 점 + "재연결 중..." |

## 7. UI 제약 규칙

### 7.1 터치 타겟
- 고객 UI 모든 터치 요소: 최소 44x44px

### 7.2 라우팅 보호
- `/admin/*`: JWT 없으면 `/admin/login` 리다이렉트
- `/customer/*`: 테이블 토큰 없으면 에러 화면

### 7.3 확인 팝업 필수

| 동작 | ConfirmDialog 메시지 |
|------|---------------------|
| 주문 삭제 | "이 주문을 삭제하시겠습니까?" |
| 이용 완료 | "테이블 {번호}의 이용을 완료하시겠습니까?" |
| 카테고리 삭제 | "삭제 시 소속 메뉴가 미분류로 이동됩니다" |
| 메뉴 삭제 | "이 메뉴를 삭제하시겠습니까?" |
| 장바구니 비우기 | "장바구니를 비우시겠습니까?" |

### 7.4 로딩 상태
- API 호출 중 해당 영역 LoadingSpinner
- 제출 버튼 비활성화 (중복 요청 방지)

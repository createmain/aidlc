| description | 선택 | - |

### 1.3 카테고리 등록/수정

| 필드 | 규칙 | 에러 메시지 |

## 3. 주문 상태 전이 규칙 (UI 반영)

### 3.1 허용 전이 맵 (버튼 활성화 기준)

| 현재 상태 | 표시할 버튼 |
|-----------|-------------|
| pending | "준비 시작" (→ preparing) |
| preparing | "준비 완료" (→ completed), "대기로 복귀" (→ pending) |
| completed | "준비중으로 복귀" (→ preparing) |

### 3.2 상태 뱃지 표시

| 상태 | 뱃지 색상 | 텍스트 |
|------|-----------|--------|
| pending | 주황/노랑 | "대기중" |
| preparing | 파랑 | "준비중" |
| completed | 초록 | "완료" |

### 3.3 삭제된 주문
- 논리 삭제된 주문(is_deleted=true)은 목록에서 표시하지 않음
- 아카이브된 주문(is_archived=true)은 현재 주문 목록에서 제외

## 4. 인증 규칙

### 4.1 관리자 JWT

| 규칙 | 동작 |
|------|------|
| 저장 위치 | localStorage 'admin_token' |
| 만료 시간 | 16시간 (서버 설정) |
| 만료 확인 | 페이지 로드 시 + API 401 응답 시 |
| 만료 시 | localStorage 삭제 → LoginPage 이동 |
| API 요청 | Authorization: Bearer {token} 헤더 |

### 4.2 테이블 토큰

| 규칙 | 동작 |
|------|------|
| 저장 위치 | localStorage 'table_token' |
| 만료 시간 | 세션 남은 시간 (최대 16시간) |
| 만료 확인 | 앱 마운트 시 + API 401 응답 시 |
| 만료 시 | 에러 화면: "세션 만료. 관리자에게 문의하세요" |
| API 요청 | Authorization: Bearer {token} 헤더 |

## 5. 에러 처리 규칙

### 5.1 에러 표시 패턴

| 에러 유형 | 표시 방식 | 자동 사라짐 |
|-----------|-----------|:-----------:|
| API 일반 에러 (네트워크, 500) | 토스트 (error) | 5초 |
| API 비즈니스 에러 (400, 409) | 토스트 (warning) | 5초 |
| 폼 검증 에러 | 인라인 (필드 하단) | 수정 시 |
| 인증 에러 (401) | 자동 로그아웃 + 리다이렉트 | - |
| 계정 잠금 (423) | 인라인 (로그인 폼) | 수정 시 |

### 5.2 Backend 에러 응답 매핑

| Backend error 코드 | Frontend 처리 |
|--------------------|---------------|
| VALIDATION_ERROR | 토스트 + details 있으면 인라인 매핑 |
| AUTH_ERROR | 인라인 에러 (로그인 폼) |
| FORBIDDEN | 토스트: "권한이 없습니다" |
| NOT_FOUND | 토스트: "데이터를 찾을 수 없습니다" |
| STATE_ERROR | 토스트: 서버 메시지 표시 |
| ACCOUNT_LOCKED | 인라인: "계정 잠금. {retryAfter}분 후 재시도" |
| INTERNAL_ERROR | 토스트: "서버 오류가 발생했습니다" |

### 5.3 토스트 규칙

| 규칙 | 값 |
|------|-----|
| 최대 동시 표시 | 3개 |
| 자동 사라짐 | success: 3초, error/warning: 5초, info: 4초 |
| 위치 | 화면 상단 우측 |
| 수동 닫기 | 항상 가능 (X 버튼) |

## 6. Realtime(SSE) 규칙

### 6.1 이벤트 처리

| 이벤트 | UI 동작 |
|--------|---------|
| new-order | 해당 테이블 카드 데이터 갱신 + 토스트("테이블 {번호}에 새 주문") + 카드 하이라이트 |
| order-status-changed | 해당 주문 상태 뱃지 갱신 |
| order-deleted | 해당 주문 목록에서 제거 + 총액 갱신 |
| session-completed | 해당 테이블 카드 리셋 (주문 0, 총액 0) |

### 6.2 하이라이트 규칙

| 규칙 | 동작 |
|------|------|
| 트리거 | new-order 이벤트 수신 시 |
| 표시 | 테이블 카드 테두리/배경 강조 |
| 해제 조건 | 관리자가 해당 테이블 카드를 클릭(확인)할 때 |
| 토스트 | 자동 사라짐 (5초) |

### 6.3 연결 상태

| 상태 | 인디케이터 | 동작 |
|------|-----------|------|
| connected | 초록 점 | 정상 수신 |
| disconnected | 빨간 점 | EventSource 자동 재연결 대기 |
| reconnecting | 노란 점 | 재연결 시도 중 |

### 6.4 재연결 후 동기화
- EventSource 재연결 성공 시 `GET /api/tables` 전체 재조회
- 재조회로 누락된 이벤트 보상 (서버 측 메시지 큐 없음, MVP 수준)

## 7. UI 제약 규칙

### 7.1 터치 타겟
- 모든 터치 가능 요소: 최소 44x44px (고객 UI)

### 7.2 라우팅 보호
- `/admin/*` 경로: JWT 없으면 `/admin/login`으로 리다이렉트
- `/customer/*` 경로: 테이블 토큰 없으면 에러 화면 표시

### 7.3 확인 팝업 필수 동작
- 주문 삭제: ConfirmDialog 필수
- 이용 완료: ConfirmDialog 필수
- 카테고리 삭제: ConfirmDialog 필수 ("소속 메뉴가 미분류로 이동됩니다")
- 메뉴 삭제: ConfirmDialog 필수
- 장바구니 비우기: ConfirmDialog 필수

### 7.4 로딩 상태
- 모든 API 호출 중 해당 영역에 LoadingSpinner 표시
- 제출 버튼 비활성화 (중복 요청 방지)
- 전체 페이지 로딩: fullPage LoadingSpinner

# 테이블오더 서비스 - 요구사항 정의서

## Intent Analysis

- **User Request**: 단일 매장용 테이블오더 서비스 MVP 구축
- **Request Type**: New Project (신규 프로젝트)
- **Scope Estimate**: Multiple Components (프론트엔드 + 백엔드 + DB)
- **Complexity Estimate**: Moderate (Realtime 통신, 세션 관리, 다중 인터페이스)

---

## 1. 프로젝트 개요

디지털 주문 시스템을 통해 고객에게는 편리한 주문 경험을, 매장 운영자에게는 효율적인 운영 환경을 제공하는 테이블오더 플랫폼.

### 1.1 핵심 결정 사항 (사용자 답변 기반)

| 항목 | 결정 |
|------|------|
| 백엔드 | Node.js + Express/Fastify (TypeScript) |
| 프론트엔드 | React (TypeScript) |
| 데이터베이스 | PostgreSQL |
| 매장 구조 | 단일 매장 전용 |
| 배포 환경 | Docker 컨테이너 기반 |
| 이미지 관리 | 서버 이미지 파일 업로드 |
| 관리자 계정 | 시스템 초기 설정 시 1개 고정 생성 |
| 동시 접속 규모 | 소규모 (10개 이하 테이블) |
| 메뉴 관리 | MVP에 포함 (CRUD 전체) |
| 프로젝트 구조 | 모노레포 |
| Security Extension | 비활성화 (프로토타입 수준) |
| PBT Extension | 비활성화 |

## 2. 기능 요구사항 (Functional Requirements)

### 2.1 고객용 기능

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 초기 설정 1회 수행 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 별도 고객 로그인 절차 없음
- 테이블 세션 기반 인증 (유효 시간: 16시간, 관리자 초기 설정 시 생성)
- 고객 별도 세션 없음 — 테이블 세션이 유일한 세션 단위

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시에도 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 주문 성공 시: 주문 번호 표시 → 장바구니 비우기 → 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 이용 완료 처리된 주문 제외
- 주문 상태 실시간 업데이트 (선택사항)
- 페이지네이션 또는 무한 스크롤

### 2.2 관리자용 기능

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한 (5회 실패 시 15분 잠금)
- 관리자 계정은 시스템 초기 설정 시 1개 고정 생성

#### FR-A02: 실시간 주문 모니터링
- Realtime 기반 주문 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상, 애니메이션)
- 2초 이내 주문 표시
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호/비밀번호 설정, 테이블 세션 생성 (유효 시간: 16시간)
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산 → 성공/실패 피드백
- **테이블 세션 처리**:
  - 이용 완료: 주문 내역을 과거 이력으로 이동, 현재 주문/총액 리셋, 세션 종료(status='completed'), 성공/실패 피드백
  - 이용 완료 후 새 고객을 위해 관리자가 테이블 재설정(새 세션 생성) 필요
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 각 주문 정보(주문 번호, 시각, 메뉴 목록, 총 금액, 매장 이용 완료 시각), 날짜 필터링, "닫기" 버튼으로 대시보드 복귀

#### FR-A04: 메뉴 관리 (MVP 포함)
- 메뉴 CRUD (등록/조회/수정/삭제)
- 카테고리 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지
- 이미지 파일 서버 업로드
- 메뉴 노출 순서 조정
- 카테고리별 조회
- 필수 필드 및 가격 범위 검증

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- Realtime 기반 주문 표시 2초 이내
- 소규모 매장 (10개 이하 테이블) 동시 접속 지원

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 터치 타겟)
- 직관적 카드 기반 레이아웃
- 명확한 시각적 계층 구조

### NFR-03: 데이터 관리
- 장바구니: 클라이언트 로컬 저장 (localStorage)
- 주문/메뉴/세션: PostgreSQL 서버 저장
- 주문 이력: OrderHistory 테이블, 세션 ID 그룹화

### NFR-04: 배포
- Docker 컨테이너 기반 배포
- 모노레포 구조 (프론트엔드 + 백엔드)

## 4. MVP 개발 범위

### 고객용
- 테이블 태블릿 자동 로그인 및 세션 관리 (FR-C01)
- 메뉴 조회 및 탐색 (FR-C02)
- 장바구니 관리 (FR-C03)
- 주문 생성 (FR-C04)
- 주문 내역 조회 (FR-C05)

### 관리자용
- 매장 인증 (FR-A01)
- 실시간 주문 모니터링 (FR-A02)
- 테이블 관리 - 초기 설정, 주문 삭제, 매장 이용 완료, 과거 내역 조회 (FR-A03)
- 메뉴 관리 - CRUD 전체 (FR-A04)

---

## 5. 제외 범위 (Out of Scope)

다음 기능은 MVP에서 구현하지 않음:
- 결제 처리 (카드, 현금, PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA, OTP)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 재고 관리, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도 API, 번역 API)

## 7. 용어 정의

| 용어 | 정의 |
|------|------|
| 테이블 세션 | 테이블 단위의 인증 및 이용 관리 단위. 관리자가 초기 설정 시 생성되며 16시간 유효. 태블릿 자동 로그인, 주문 생성, 주문 내역 조회 등 모든 테이블 활동의 기준이 됨. 이용 완료 처리 시 해당 세션의 주문 이력이 과거로 이동하고 세션이 종료됨(status='completed'). 새 고객을 위해 관리자가 테이블 재설정(새 세션 생성) 필요. 세션 만료 시 관리자 재설정 필요. 고객 별도 세션은 존재하지 않으며, 테이블 세션이 유일한 세션 단위임. |
| MVP | Minimum Viable Product, 최소 기능 제품 |
| Realtime | 서버에서 클라이언트로의 실시간 데이터 전송 기능. 구현 기술로 SSE(Server-Sent Events)를 사용 |

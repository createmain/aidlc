# 테이블오더 서비스 - User Stories

> **분류**: User Journey 기반 | **세분화**: 중간 단위 | **AC**: Given-When-Then | **우선순위**: MoSCoW

## 고객 Journey (Persona: 김고객)

### US-C01: 테이블 태블릿 자동 로그인
**Persona**: 김고객
**Priority**: Must
**Requirement**: FR-C01

**As a** 고객, **I want** 테이블 태블릿이 자동으로 로그인되어 있기를, **so that** 별도 인증 절차 없이 즉시 메뉴를 볼 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 관리자가 태블릿 초기 설정을 완료한 상태에서,
  - **When** 태블릿 브라우저를 열면,
  - **Then** 저장된 인증 정보로 자동 로그인되어 메뉴 화면이 표시된다.

- AC2
  - **Given** 자동 로그인에 성공한 상태에서,
  - **When** 브라우저를 새로고침하면,
  - **Then** 로그인 상태가 유지되어 메뉴 화면이 다시 표시된다.

- AC3
  - **Given** 저장된 인증 정보가 유효하지 않을 때,
  - **When** 태블릿 브라우저를 열면,
  - **Then** 로그인 실패 메시지가 표시되고 관리자 재설정이 필요함을 안내한다.

---

### US-C02: 메뉴 조회 및 탐색
**Persona**: 김고객
**Priority**: Must
**Requirement**: FR-C02

**As a** 고객, **I want** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를, **so that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 메뉴 화면(홈)에 진입한 상태에서,
  - **When** 화면을 보면,
  - **Then** 카테고리별로 분류된 메뉴가 카드 형태로 표시되며 메뉴명, 가격, 이미지가 보인다.

- AC2
  - **Given** 메뉴 목록이 표시된 상태에서,
  - **When** 특정 카테고리를 선택하면,
  - **Then** 해당 카테고리의 메뉴만 필터링되어 표시된다.

- AC3
  - **Given** 메뉴 카드가 표시된 상태에서,
  - **When** 메뉴 카드를 터치하면,
  - **Then** 메뉴명, 가격, 설명, 이미지가 포함된 상세 정보가 표시된다.

- AC4
  - **Given** 메뉴 화면에서,
  - **When** 터치 가능한 요소를 확인하면,
  - **Then** 모든 터치 타겟이 최소 44x44px 이상이다.

### US-C03: 장바구니 관리
**Persona**: 김고객
**Priority**: Must
**Requirement**: FR-C03

**As a** 고객, **I want** 주문할 메뉴를 장바구니에 담고 수량을 조절하기를, **so that** 주문 전에 원하는 메뉴와 수량을 정리할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 메뉴 상세 화면에서,
  - **When** 장바구니 추가 버튼을 터치하면,
  - **Then** 해당 메뉴가 장바구니에 추가되고 총 금액이 업데이트된다.

- AC2
  - **Given** 장바구니에 메뉴가 있는 상태에서,
  - **When** 수량 증가/감소 버튼을 터치하면,
  - **Then** 수량이 변경되고 총 금액이 실시간으로 재계산된다.

- AC3
  - **Given** 장바구니에 메뉴가 있는 상태에서,
  - **When** 특정 메뉴를 삭제하면,
  - **Then** 해당 메뉴가 장바구니에서 제거되고 총 금액이 재계산된다.

- AC4
  - **Given** 장바구니에 메뉴가 있는 상태에서,
  - **When** 장바구니 비우기를 실행하면,
  - **Then** 모든 메뉴가 제거되고 총 금액이 0원이 된다.

- AC5
  - **Given** 장바구니에 메뉴가 있는 상태에서,
  - **When** 브라우저를 새로고침하면,
  - **Then** 장바구니 내용이 유지된다 (localStorage).

---

### US-C04: 주문 생성
**Persona**: 김고객
**Priority**: Must
**Requirement**: FR-C04

**As a** 고객, **I want** 장바구니의 메뉴를 최종 확인하고 주문을 확정하기를, **so that** 실제 주문이 매장에 전달된다.

#### Acceptance Criteria

- AC1
  - **Given** 장바구니에 메뉴가 있는 상태에서,
  - **When** 주문 확정 버튼을 터치하면,
  - **Then** 주문 내역(메뉴명, 수량, 단가, 총 금액)이 최종 확인 화면에 표시된다.

- AC2
  - **Given** 최종 확인 화면에서,
  - **When** 주문을 확정하면,
  - **Then** 주문이 서버에 전송되고 주문 번호가 표시된다. 주문 정보에는 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID가 포함된다.

- AC3
  - **Given** 주문이 성공한 상태에서,
  - **When** 주문 번호가 표시되면,
  - **Then** 장바구니가 비워지고 5초 후 메뉴 화면으로 자동 리다이렉트된다.

- AC4
  - **Given** 주문 전송 중 오류가 발생했을 때,
  - **When** 서버 응답이 실패이면,
  - **Then** 에러 메시지가 표시되고 장바구니 내용은 유지된다.

---

### US-C05: 주문 내역 조회
**Persona**: 김고객
**Priority**: Must
**Requirement**: FR-C05

**As a** 고객, **I want** 현재 세션의 주문 내역과 상태를 확인하기를, **so that** 내가 주문한 메뉴의 진행 상황을 알 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 현재 테이블 세션에 주문이 있는 상태에서,
  - **When** 주문 내역 화면에 진입하면,
  - **Then** 현재 세션의 주문만 시간 순으로 표시된다 (이전 세션 주문 제외).

- AC2
  - **Given** 주문 내역이 표시된 상태에서,
  - **When** 각 주문을 확인하면,
  - **Then** 주문 번호, 주문 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)가 표시된다.

- AC3
  - **Given** 이용 완료 처리된 주문이 있을 때,
  - **When** 주문 내역을 조회하면,
  - **Then** 이용 완료된 주문은 목록에서 제외된다.

- AC4
  - **Given** 주문 내역이 많을 때,
  - **When** 목록을 스크롤하면,
  - **Then** 페이지네이션 또는 무한 스크롤로 추가 주문이 로드된다.

---

### US-C06: 주문 상태 실시간 업데이트 (선택사항)
**Persona**: 김고객
**Priority**: Could
**Requirement**: FR-C05 (선택사항)

**As a** 고객, **I want** 주문 상태가 변경되면 실시간으로 화면에 반영되기를, **so that** 새로고침 없이 주문 진행 상황을 확인할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 주문 내역 화면을 보고 있는 상태에서,
  - **When** 관리자가 주문 상태를 변경하면,
  - **Then** 해당 주문의 상태가 새로고침 없이 실시간으로 업데이트된다.

## 관리자 Journey (Persona: 박관리자)

### US-A00: 시스템 초기 설정 (관리자 계정 생성)
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A01

**As a** 관리자, **I want** 시스템 최초 구동 시 관리자 계정이 자동으로 생성되기를, **so that** 별도 회원가입 없이 바로 매장 관리를 시작할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 시스템이 최초로 구동될 때,
  - **When** 초기 설정 프로세스가 실행되면,
  - **Then** 1개의 관리자 계정(매장 식별자, 사용자명, 비밀번호)이 자동으로 생성된다.

- AC2
  - **Given** 관리자 계정이 이미 존재하는 상태에서,
  - **When** 시스템이 재시작되면,
  - **Then** 중복 계정이 생성되지 않고 기존 계정이 유지된다.

- AC3
  - **Given** 초기 생성된 관리자 계정에서,
  - **When** 비밀번호 저장 방식을 확인하면,
  - **Then** bcrypt로 해싱되어 저장되어 있다.

---

### US-A01: 매장 인증 (로그인)
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A01

**As a** 관리자, **I want** 매장 식별자와 계정 정보로 로그인하기를, **so that** 매장 관리 시스템에 안전하게 접근할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 관리자 로그인 화면에서,
  - **When** 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인하면,
  - **Then** JWT 토큰이 발급되고 관리자 대시보드로 이동한다.

- AC2
  - **Given** 로그인에 성공한 상태에서,
  - **When** 브라우저를 새로고침하면,
  - **Then** 세션이 유지되어 재로그인 없이 대시보드가 표시된다.

- AC3
  - **Given** 로그인 후 16시간이 경과했을 때,
  - **When** 다음 요청을 보내면,
  - **Then** 자동 로그아웃되어 로그인 화면으로 이동한다.

- AC4
  - **Given** 잘못된 인증 정보를 입력했을 때,
  - **When** 로그인을 시도하면,
  - **Then** 에러 메시지가 표시되고 5회 실패 시 15분간 로그인이 잠금된다.

---

### US-A02: 실시간 주문 모니터링
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A02

**As a** 관리자, **I want** 테이블별 주문 현황을 대시보드에서 확인하기를, **so that** 매장 전체 주문 상황을 한눈에 파악할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 관리자 대시보드에 진입한 상태에서,
  - **When** 화면을 보면,
  - **Then** 테이블별 카드가 그리드 레이아웃으로 표시되며 각 카드에 총 주문액과 최신 주문 미리보기가 보인다.

- AC2
  - **Given** 대시보드에서 테이블 카드가 표시된 상태에서,
  - **When** 특정 테이블 카드를 클릭하면,
  - **Then** 해당 테이블의 전체 주문 메뉴 목록 상세가 표시된다.

- AC3
  - **Given** 대시보드에서,
  - **When** 주문 상태 변경 버튼(대기중→준비중→완료)을 클릭하면,
  - **Then** 해당 주문의 상태가 변경되고 화면에 반영된다.

- AC4
  - **Given** 대시보드에서,
  - **When** 테이블별 필터를 적용하면,
  - **Then** 선택한 테이블의 주문만 표시된다.

---

### US-A03: 실시간 신규 주문 알림
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A02 (실시간 통신)

**As a** 관리자, **I want** 신규 주문이 들어오면 실시간으로 대시보드에 반영되기를, **so that** 주문을 놓치지 않고 즉시 확인할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 대시보드를 보고 있는 상태에서,
  - **When** 고객이 새 주문을 생성하면,
  - **Then** 2초 이내에 해당 테이블 카드에 신규 주문이 표시된다.

- AC2
  - **Given** 신규 주문이 도착했을 때,
  - **When** 대시보드를 확인하면,
  - **Then** 신규 주문이 시각적으로 강조(색상 변경, 애니메이션)되어 표시된다.

- AC3
  - **Given** Realtime 연결이 끊어졌을 때,
  - **When** 네트워크가 복구되면,
  - **Then** Realtime 연결이 자동으로 재수립되고 누락된 주문이 반영된다.

---

### US-A04: 테이블 초기 설정
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A03

**As a** 관리자, **I want** 테이블 태블릿의 번호와 비밀번호를 설정하기를, **so that** 고객이 해당 테이블에서 자동 로그인하여 주문할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 테이블 관리 화면에서,
  - **When** 테이블 번호와 비밀번호를 입력하고 설정을 저장하면,
  - **Then** 테이블 세션이 생성(유효 시간: 16시간)되고 해당 태블릿에서 자동 로그인이 활성화된다.

- AC2
  - **Given** 테이블 설정이 완료된 상태에서,
  - **When** 해당 태블릿에서 브라우저를 열면,
  - **Then** 설정된 정보로 자동 로그인되어 메뉴 화면이 표시된다.

---

### US-A05: 주문 삭제
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A03

**As a** 관리자, **I want** 특정 주문을 삭제하기를, **so that** 잘못된 주문이나 취소된 주문을 정리할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 테이블 주문 상세에서,
  - **When** 특정 주문의 삭제 버튼을 클릭하면,
  - **Then** 확인 팝업이 표시된다.

- AC2
  - **Given** 삭제 확인 팝업에서,
  - **When** 확인을 클릭하면,
  - **Then** 주문이 즉시 삭제되고 테이블 총 주문액이 재계산되며 성공 피드백이 표시된다.

- AC3
  - **Given** 삭제 처리 중 오류가 발생했을 때,
  - **When** 서버 응답이 실패이면,
  - **Then** 실패 피드백이 표시되고 주문은 유지된다.

---

### US-A06: 테이블 세션 종료 (이용 완료)
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A03

**As a** 관리자, **I want** 테이블 이용 완료 처리를 하기를, **so that** 해당 테이블의 주문이 과거 이력으로 이동하고 새 고객을 받을 준비가 된다.

#### Acceptance Criteria

- AC1
  - **Given** 활성 세션이 있는 테이블에서,
  - **When** 이용 완료 버튼을 클릭하면,
  - **Then** 확인 팝업이 표시된다.

- AC2
  - **Given** 이용 완료 확인 팝업에서,
  - **When** 확인을 클릭하면,
  - **Then** 현재 세션의 주문 내역이 과거 이력으로 이동하고, 테이블 현재 주문 목록과 총 주문액이 0으로 리셋되며, 성공 피드백이 표시된다.

- AC3
  - **Given** 이용 완료 처리 중 오류가 발생했을 때,
  - **When** 서버 응답이 실패이면,
  - **Then** 실패 피드백이 표시되고 세션은 유지된다.

- AC4
  - **Given** 이용 완료 처리 후,
  - **When** 새 고객이 해당 테이블에서 주문을 하면,
  - **Then** 동일 테이블 세션 내에서 주문이 생성된다 (세션 만료 전까지 별도 세션 생성 없음).

---

### US-A07: 과거 주문 내역 조회
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A03

**As a** 관리자, **I want** 테이블별 과거 주문 내역을 조회하기를, **so that** 이전 고객의 주문 이력을 확인하고 매장 운영에 참고할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 테이블 관리 화면에서,
  - **When** 과거 내역 버튼을 클릭하면,
  - **Then** 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다.

- AC2
  - **Given** 과거 주문 목록에서,
  - **When** 각 주문을 확인하면,
  - **Then** 주문 번호, 시각, 메뉴 목록, 총 금액, 매장 이용 완료 시각이 표시된다.

- AC3
  - **Given** 과거 주문 목록에서,
  - **When** 날짜 필터를 적용하면,
  - **Then** 선택한 날짜 범위의 주문만 표시된다.

- AC4
  - **Given** 과거 주문 내역 화면에서,
  - **When** 닫기 버튼을 클릭하면,
  - **Then** 대시보드로 복귀한다.

---

### US-A09: 테이블 목록 조회 및 관리
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A03

**As a** 관리자, **I want** 매장의 전체 테이블 목록과 각 테이블의 현재 상태를 확인하기를, **so that** 테이블별 운영 현황을 파악하고 관리 작업을 수행할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 테이블 관리 화면에 진입한 상태에서,
  - **When** 화면을 보면,
  - **Then** 매장에 등록된 전체 테이블 목록이 표시되며 각 테이블의 번호, 세션 상태(활성/만료), 현재 주문 건수가 보인다.

- AC2
  - **Given** 테이블 목록에서 특정 테이블을 선택한 상태에서,
  - **When** 테이블 상세를 확인하면,
  - **Then** 해당 테이블의 현재 주문 목록, 총 주문액, 세션 정보가 표시된다.

---

### US-A08: 메뉴 및 카테고리 관리 (CRUD)
**Persona**: 박관리자
**Priority**: Must
**Requirement**: FR-A04

**As a** 관리자, **I want** 메뉴와 카테고리를 등록, 조회, 수정, 삭제하기를, **so that** 매장 메뉴를 동적으로 관리할 수 있다.

#### Acceptance Criteria

- AC1
  - **Given** 메뉴 관리 화면에서,
  - **When** 새 메뉴 등록 폼에 메뉴명, 가격, 설명, 카테고리를 입력하고 이미지 파일을 서버에 업로드하여 저장하면,
  - **Then** 메뉴가 등록되고 목록에 표시된다.

- AC2
  - **Given** 메뉴 관리 화면에서,
  - **When** 카테고리를 선택하면,
  - **Then** 해당 카테고리의 메뉴만 필터링되어 표시된다.

- AC3
  - **Given** 기존 메뉴를 선택한 상태에서,
  - **When** 정보를 수정하고 저장하면,
  - **Then** 변경 사항이 반영되고 고객 메뉴 화면에도 업데이트된다.

- AC4
  - **Given** 기존 메뉴를 선택한 상태에서,
  - **When** 삭제를 실행하면,
  - **Then** 메뉴가 삭제되고 목록에서 제거된다.

- AC5
  - **Given** 메뉴 등록/수정 시,
  - **When** 필수 필드가 비어있거나 가격이 유효 범위를 벗어나면,
  - **Then** 검증 에러 메시지가 표시되고 저장이 차단된다.

- AC6
  - **Given** 메뉴 관리 화면에서,
  - **When** 메뉴 노출 순서를 변경하면,
  - **Then** 변경된 순서가 저장되고 고객 메뉴 화면에 반영된다.

- AC7
  - **Given** 카테고리 관리 화면에서,
  - **When** 새 카테고리명을 입력하고 등록하면,
  - **Then** 카테고리가 생성되고 카테고리 목록에 표시된다.

- AC8
  - **Given** 기존 카테고리를 선택한 상태에서,
  - **When** 카테고리명을 수정하고 저장하면,
  - **Then** 변경 사항이 반영되고 해당 카테고리에 속한 메뉴의 분류도 유지된다.

- AC9
  - **Given** 기존 카테고리를 선택한 상태에서,
  - **When** 삭제를 실행하면,
  - **Then** 카테고리가 삭제되고 목록에서 제거된다.

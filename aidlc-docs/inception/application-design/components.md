# 테이블오더 서비스 - 컴포넌트 정의

---

## 1. Frontend (단일 React 앱)

### 1.1 Customer UI
- **목적**: 고객이 메뉴 탐색, 장바구니 관리, 주문 생성, 주문 내역 조회를 수행하는 인터페이스
- **책임**:
  - 태블릿 자동 로그인 및 세션 유지
  - 카테고리별 메뉴 표시 (카드 레이아웃)
  - 장바구니 로컬 관리 (localStorage)
  - 주문 생성 및 결과 처리
  - 주문 성공 시 주문 번호 표시 → 장바구니 비우기 → 5초 후 메뉴 화면 자동 리다이렉트
  - 주문 실패 시 에러 메시지 표시 및 장바구니 유지
  - 현재 세션 주문 내역 조회
- **경로**: `/customer/*`

### 1.2 Admin UI
- **목적**: 관리자가 주문 모니터링, 테이블 관리, 메뉴 관리를 수행하는 인터페이스
- **책임**:
  - 관리자 로그인 (JWT)
  - 실시간 주문 대시보드 (Realtime 수신)
  - 주문 상태 변경
  - 테이블 초기 설정 / 세션 종료 / 주문 삭제
  - 과거 주문 내역 조회
  - 메뉴 CRUD + 이미지 업로드
- **경로**: `/admin/*`

### 1.3 Shared Components
- **목적**: 고객/관리자 공통 UI 요소
- **책임**: 공통 레이아웃, 에러 처리, 로딩 상태, API 클라이언트, 인증 컨텍스트

---

## 2. Backend (Node.js + Express/Fastify)

### 2.1 Auth Module
- **목적**: 관리자 인증 및 테이블 태블릿 인증
- **책임**: 관리자 로그인(JWT 발급), 테이블 태블릿 인증, 토큰 검증 미들웨어, 로그인 시도 제한
- **로그인 시도 제한 저장**: login_attempts DB 테이블 (identifier, attempt_count, last_attempt_at, locked_until) — 5회 실패 시 15분 잠금. 식별자: "{username}"

### 2.2 Menu Module
- **목적**: 메뉴 및 카테고리 데이터 관리
- **책임**: 메뉴 CRUD, 카테고리 CRUD, 카테고리별 조회, 이미지 파일 업로드/서빙, 노출 순서 관리, 입력 검증

### 2.3 Order Module
- **목적**: 주문 생성 및 상태 관리
- **책임**: 주문 생성, 주문 상태 변경(대기중→준비중→완료), 주문 삭제, 테이블별/세션별 주문 조회, 실시간 이벤트 발행

### 2.4 Table Module
- **목적**: 테이블 및 세션 라이프사이클 관리
- **책임**: 테이블 초기 설정, 세션 시작/종료, 이용 완료 처리(주문→이력 이동), 과거 주문 내역 조회

### 2.5 Realtime Module
- **목적**: 실시간 통신 관리
- **책임**: 실시간 연결 관리, 이벤트 브로드캐스트(신규 주문, 상태 변경), 연결 해제/재연결 처리

### 2.6 System Module
- **목적**: 시스템 초기 설정
- **책임**: 최초 구동 시 관리자 계정 자동 생성, 중복 생성 방지

---

## 3. Database (PostgreSQL)

- **목적**: 매장 설정, 테이블, 메뉴, 주문, 세션, 주문 이력 데이터 영속화
- **주요 테이블**: settings, admin_users, tables, table_sessions, categories, menu_items, orders, order_items, order_history, login_attempts
